Main Integration Blockers
1. The Auth Hierarchy

OAuth2 and API Key dominate the SaaS landscape. OAuth2 represents 60% of systems, serving as the standard for user-facing productivity applications (Slack, Notion, Google Workspace). API Keys represent 42%, dominating developer platforms, SEO toolsets, and infrastructure (Apify, Firecrawl, Supabase). A small subset (8%) of legacy systems still require Basic Auth or Custom tokens.

2. Category Gating Correlations

There is a strict correlation between categories and developer gates. Developer Tools (Category 7) and Productivity (Category 8) are almost 100% self-serve with free tiers, making them "easy wins." Conversely, Finance (Category 9) and AI/Research (Category 10) are highly gated, often requiring paid subscriptions (Ahrefs, Brex), manual admin approvals (Google Ads, Amazon), or sales partnership onboarding (DealCloud, iPayX, NotebookLM).

3. The Buildability Verdict

79% of audited applications are buildable today (Yes: 68%, Partial: 11%), meaning developer API interfaces exist. The remaining 21% represent a hard blocker where no public API console exists (NotebookLM, Otter AI, fanbasis) or integrations are strictly locked behind enterprise sales partnerships, highlighting the need for business-level partnerships.

How it was built

The Agent Architecture & Human Interface

Research Agent Logic & Flow

To audit all 100 applications systematically, we built a Python-based agentic pipeline utilizing Gemini 3.5 Flash and Google Search Grounding. Search grounding enables the model to issue dynamic queries, retrieve fresh web pages, and resolve structured JSON metadata against active developer documents.

1 Structured Input Setup

The catalog of 100 apps mapped from Notion screenshots was structured in apps_list.json to provide seeds for the research query.

2 Search Grounding Execution

Gemini was configured with Google Search tools and structured Pydantic schemas, enforcing precise constraints on the schema values.

3 Resilient Local Caching

Results are cached incrementally. If an API rate limit (429) occurs or connection drops, progress is preserved and resume-friendly.

Where Humans Were Needed

AI agents excel at horizontal data gathering but can make systematic errors due to developer portal signup semantics. A human-in-the-loop audit was crucial in the following areas:

  • 1
    Resolving Gated Subscriptions
    The agent misclassified tools like Ahrefs and SE Ranking as self-serve because signups are public, but human verification confirmed that API credentials are restricted to high-tier paid developer subscriptions.
  • 2
    Auditing Vetting Requirements
    For Amazon SP-API and Google Ads, the agent missed that developers must undergo manual business vetting and profile review processes before receiving live keys. Humans corrected this to "Gated (Admin Approval)".
  • 3
    Validating Evidence URLs
    The agent occasionally cited general homepage links instead of precise developer subdomains. The verification loop resolved this to direct developer URLs.
